# Daily Brief (prototype)\n\nThis service generates a personalized daily brief composed of weather, news, and calendar events.\n\n## Features\n- Weather from Open-Meteo (no key).\n- News from NewsAPI (or RSS fallback).\n- Calendar via public ICS URL (or mocked events). Google Calendar OAuth can be added for per-user calendars.\n- Endpoint: `GET /brief?lat=...&lon=...&news=...&maxHeadlines=...`\n\n## Quickstart\n1. Copy `.env.example` to `.env` and fill values.\n2. Install:\n```\nnpm install\n```\n3. Run:\n```\nnpm start\n```\n4. Example:\n```\nGET http://localhost:3000/brief?lat=37.7749&lon=-122.4194&news=technology&maxHeadlines=3\n```\n\n## Production considerations\n- Add user authentication and store profiles (lat/lon, preferences).\n- Use Google OAuth for per-user Calendar access and securely store refresh tokens.\n- Rate-limit and cache upstream API responses (Redis).\n- Add templating for email/SMS/push delivery (SendGrid, Twilio, FCM/APNs).\n- Add graceful retries and circuit-breakers for flaky upstreams.\n- Add tests, CI, and observability (metrics/logs/tracing).\n\n## Extensibility\n- Add more sources (stocks, commute, to-dos).\n- Use a rules engine or ML model for personalization.\n- Provide multi-channel delivery and per-user scheduling.
